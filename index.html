Top 50 Operating System Interview Questions

1
What is an Operating System?
System software that manages hardware and software. Acts as interface between user and hardware. Handles process, memory, files, devices. Examples: Windows, Linux, macOS.
2
Main Functions of OS
Manages process, memory, files, I/O. Provides security and UI. Ensures efficient resource usage. Handles error recovery.
3
What is a Process?
Program in execution. Contains code, data, state. Has own resources. Scheduled by OS.
4
What is a Thread?
Lightweight process unit. Multiple threads per process. Shares resources. Improves performance.
5
Process vs Thread
Process: Own memory, heavy. Thread: Shared memory, light. Thread switching is faster.
6
What is Multitasking?
Multiple programs run together. CPU switches rapidly. Improves efficiency. Used in modern OS.
7
What is Multiprocessing?
Uses multiple CPUs/cores. Parallel execution. High performance. Used in servers.
8
What is Multithreading?
Multiple threads in process. Runs concurrently. Better CPU utilization. Used in browsers.
9
What is Scheduling?
Decides next process execution. Handled by scheduler. Improves CPU efficiency. Uses algorithms.
10
Scheduling Algorithm
Decides execution order. Examples: FCFS, SJF, RR. Affects waiting & turnaround. Ensures fairness.
11
FCFS Scheduling
First come first serve. Simple to implement. Long waiting time. Not efficient always.
12
SJF Scheduling
Shortest job executes first. Minimum average waiting. Preemptive/non-preemptive. Burst time hard to predict.
13
Round Robin
Fixed time slice. Equal CPU time. No starvation. Time-sharing systems.
14
Priority Scheduling
Based on priority. Higher priority runs first. May cause starvation.
15
Starvation
Process never gets CPU. Low priority suffers. Solved using aging.
16
Aging
Priority increases over time. Prevents starvation. Ensures fairness.
17
Deadlock
Processes wait forever. Resources never released. System halts.
18
Deadlock Conditions
Mutual exclusion. Hold & wait. No preemption. Circular wait.
19
Deadlock Prevention
Avoid one condition. Restricts resources. Safe but inefficient.
20
Deadlock Avoidance
Predicts deadlock. Uses Banker’s algorithm. Needs prior info.
21
Banker’s Algorithm
Avoids deadlock. Checks safe state. Needs max resource info.
22
Memory Management
Allocates/deallocates memory. Avoids leaks. Ensures efficiency.
23
Paging
Fixed-size pages. No external fragmentation. Uses page table.
24
Segmentation
Logical segments. Variable size. Matches program logic.
25
Paging vs Segmentation
Paging: fixed blocks. Segmentation: variable blocks. Paging avoids external fragmentation.
26
Virtual Memory
Uses disk as RAM. Runs large programs. Implemented using paging.
27
Thrashing
Excessive swapping. Low CPU usage. Due to low memory.
28
Page Fault
Page not in memory. Loaded from disk. Increases execution time.
29
Fragmentation
Wasted memory. Internal & external types. Reduces efficiency.
30
File System
Organizes data. Manages files/directories. Example: NTFS, FAT.
31
Directory
Stores file info. Organizes files. Improves access.
32
I/O Management
Manages devices. Uses drivers. Handles buffering/caching.
33
Buffering
Temporary storage. Smooth data flow. Improves speed.
34
Spooling
Disk-based buffering. Used for printers. Improves CPU use.
35
Context Switching
Saves process state. Switches CPU. Adds overhead.
36
Kernel
Core of OS. Controls hardware. Runs in privileged mode.
37
Types of Kernel
Monolithic. Microkernel. Hybrid.
38
System Call
Interface to OS services. Runs in kernel mode. File & process ops.
39
User vs Kernel Mode
User: limited access. Kernel: full access. Ensures safety.
40
Semaphore
Synchronization tool. Controls shared resources. Prevents race condition.
41
Mutex
Mutual exclusion. One thread at a time. Used in critical section.
42
Synchronization
Coordinates processes. Avoids inconsistency. Essential in concurrency.
43
Race Condition
Simultaneous access. Unpredictable result. Needs synchronization.
44
Critical Section
Shared resource code. One process at a time. Uses lock/mutex.
45
Booting
System startup. Loads OS. Cold & warm boot.
46
Interrupt
Stops current task. Handles urgent events. Hardware/software.
47
Polling
Repeated checking. CPU waits actively. Inefficient.
48
Time Sharing System
Multiple users. CPU time sliced. Responsive system.
49
Distributed OS
Multiple systems. Appears single. High reliability.
50
Why OS Important?
Manages resources. Provides UI. Ensures security & stability.
